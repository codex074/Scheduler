# Pharmacy Shift Scheduler

> ระบบจัดตารางเวรห้องยารายเดือนแบบอัตโนมัติ พร้อมหน้าจอภาษาไทย, ตรวจสอบกฎเวรอัตโนมัติ, แก้ไขตารางรายช่อง, และส่งออก Excel ได้ทันที

## ภาพรวม

โปรเจกต์นี้เป็นเว็บแอปสำหรับจัดตารางเวรรายเดือนของทีมงานห้องยา โดยระบบจะ:

- สร้างตารางเวรอัตโนมัติจากกฎจริงของหน่วยงาน
- จำกัดเวรตามอายุ, สถานะตั้งครรภ์, และเวรที่อนุญาตเฉพาะบุคคล
- ป้องกันการลงเวรชนกันหรือผิดเงื่อนไข
- ปรับสมดุลภาระงานให้ใกล้เคียงกันที่สุด
- รองรับการ `Import` / `Export` ข้อมูลเพื่อทำงานต่อใน Excel

## วิธีเริ่มต้นแบบเร็วที่สุด

ถ้าต้องการใช้งานแบบ `clone แล้วเปิดได้เลย` แนะนำให้ใช้ Docker

## จุดเด่น

- `Generate ตาราง` ด้วย greedy + backtracking + random restarts
- รองรับ `ทีมงาน`, `วันหยุด`, `ตารางเวร`, และ `ประวัติ`
- แก้ไขเวรรายคนรายวันจากหน้าเว็บได้ทันที
- ตรวจสอบ hard constraints อัตโนมัติก่อนบันทึก
- ส่งออก `.xlsx` เป็น 2 sheet: `ตารางเวร` และ `สรุป`
- นำเข้าตารางจาก Excel กลับเข้าระบบได้

## เริ่มต้นใช้งาน

### แบบที่แนะนำ: ใช้งานผ่าน Docker

#### 1. Clone โปรเจกต์

```bash
git clone https://github.com/codex074/Scheduler.git
cd Scheduler
```

#### 2. สร้างและเปิด container

```bash
docker compose up --build
```

ระบบจะทำสิ่งต่อไปนี้ให้อัตโนมัติ:

- build image
- ติดตั้ง dependencies
- build Next.js app
- รัน Prisma migrations
- สร้างฐานข้อมูล SQLite ถ้ายังไม่มี
- เปิดแอปที่ port `3000`

#### 3. เข้าใช้งาน

เปิด [http://localhost:3000](http://localhost:3000)

#### 4. รันแบบ background

```bash
docker compose up --build -d
```

#### 5. ดู log

```bash
docker compose logs -f
```

#### 6. หยุดการทำงาน

```bash
docker compose down
```

#### 7. ลบข้อมูลทั้งหมดและเริ่มใหม่

คำสั่งนี้จะลบฐานข้อมูลที่เก็บอยู่ใน Docker volume ด้วย

```bash
docker compose down -v
```

#### 8. อัปเดตโปรเจกต์หลัง pull โค้ดใหม่

```bash
git pull
docker compose up --build -d
```

### ตำแหน่งข้อมูลเมื่อใช้ Docker

- ข้อมูล SQLite จะถูกเก็บใน Docker volume ชื่อ `scheduler_data`
- ต่อให้ลบ container แล้วสร้างใหม่ ข้อมูลจะยังอยู่ ถ้าไม่สั่ง `docker compose down -v`

### ไฟล์ Docker ที่เพิ่มในโปรเจกต์

| ไฟล์ | หน้าที่ |
| --- | --- |
| `Dockerfile` | สร้าง image สำหรับ production use |
| `docker-compose.yml` | สั่ง build + run พร้อม volume และ port |
| `docker/entrypoint.sh` | รัน migrations ก่อน start app |
| `.dockerignore` | กันไฟล์ไม่จำเป็นไม่ให้ถูก copy เข้า image |

เปิดที่ [http://localhost:3000](http://localhost:3000)

## ลำดับการใช้งานที่แนะนำ

1. ไปที่ `ทีมงาน` แล้วเพิ่มสมาชิกให้ครบ
2. ไปที่ `วันหยุด` แล้วบันทึกวันหยุดราชการของปีนั้น
3. ไปที่ `ตารางเวร` แล้วสร้างเดือนที่ต้องการ
4. กด `Generate ตาราง`
5. ตรวจสอบผลลัพธ์และแก้ไขรายช่องหากจำเป็น
6. กด `Export Excel` หากต้องการไฟล์ใช้งานต่อ
7. กด `บันทึกยืนยัน` เมื่อตารางสมบูรณ์

## Rules การจัดตารางเวร

### กฎสิทธิ์การรับเวรอัตโนมัติ

ระบบจะคำนวณว่าแต่ละคน "ทำเวรอะไรได้บ้าง" จากลำดับนี้:

1. ถ้ามี `allowedShifts` แบบกำหนดเอง ระบบจะใช้ค่านั้นทันที
2. ถ้าไม่มี manual override ระบบจะดู `pregnancyStatus`
3. ถ้าไม่มีสถานะตั้งครรภ์ ระบบจะดู `อายุ`

> ถ้าตั้งค่า `เวรที่ทำได้` แบบกำหนดเองในหน้า `ทีมงาน` กฎอายุและการตั้งครรภ์จะถูกข้ามไป

### สิทธิ์ตามสถานะตั้งครรภ์

| สถานะ | เช้า | บ่าย | ดึก | เวรเสริม/รุ่งเช้า |
| --- | --- | --- | --- | --- |
| `early` | ได้ | ได้ | ได้ | ได้ |
| `mid` | ได้ | ไม่ได้ | ไม่ได้ | ไม่ได้ |
| `late` | ไม่ได้ | ไม่ได้ | ไม่ได้ | ไม่ได้ |
| `postpartum` | ได้ | ได้ | ได้ | ได้ |

### สิทธิ์ตามอายุ

| กลุ่มอายุ | ทำเวรได้ |
| --- | --- |
| `>=50` | ไม่รับเวรอัตโนมัติ |
| `47-49` | รับได้เฉพาะ `morning-SURG` |
| `45-46` | รับเวร `เช้า` และ `บ่าย` |
| `33-44` | รับได้ทุกกลุ่มเวร |
| `<33` | รับได้ทุกกลุ่มเวร |

### Hard Constraints

กฎเหล่านี้ "ห้ามผิด" ทั้งตอน Generate และตอนแก้ไขรายช่อง

| รหัส | ความหมาย |
| --- | --- |
| `HC1` | วันเดียวกันห้ามมี `ดึก` คู่กับ `รุ่งเช้า` |
| `HC2` | วันเดียวกันห้ามมี `ดึก` คู่กับ `เช้า` |
| `HC3` | ห้ามลง `บ่าย` ของวัน D แล้วต่อ `ดึก` ของวัน D+1 |
| `HC4` | ห้ามมีเวรซ้อนเวลากันในวันเดียวกัน |
| `HC5` | ต้องผ่านสิทธิ์เวรตามอายุ/การตั้งครรภ์/allowedShifts |

### Soft Constraints

กฎกลุ่มนี้มีไว้ "ช่วยให้ตารางสมดุลขึ้น" ถ้าหลีกเลี่ยงได้ระบบจะพยายามหลีกเลี่ยง

| รหัส | สิ่งที่ระบบพยายามลด |
| --- | --- |
| `SC1` | เวรหมวดเดียวกันติดกัน 3 ครั้งขึ้นไป |
| `SC2` | ระยะห่างระหว่างเวรของคนเดียวกันไม่สม่ำเสมอเกินไป |
| `SC3` | เวรวันหยุดกระจุกที่บางคนมากเกินไป |
| `SC5` | เวรเสาร์-อาทิตย์กระจายไม่สมดุล |
| `SC6` | เวรดึกติดกันข้ามวัน |
| `SC7` | ภาระงานรวมในกลุ่มอายุ/กลุ่มตั้งครรภ์ต่างกันมากเกินไป |

> หมายเหตุ: ระบบยังเก็บช่อง `SC4` ไว้ในโครงสร้างคะแนน แต่ logic หลักที่ใช้งานจริงตอนนี้คือ `SC1`, `SC2`, `SC3`, `SC5`, `SC6`, `SC7`

## วิธีใช้งานแต่ละหน้า

### ทีมงาน

เพิ่มข้อมูลหลักของสมาชิก:

- `nickname`
- `phaId`
- `dateOfBirth`
- `pregnancyStatus`
- `isActive`
- `allowedShifts` แบบกำหนดเองถ้าต้องการ

สิ่งสำคัญ:

- สมาชิกที่ `ปิดใช้งาน` จะไม่ถูกใช้ตอน Generate
- ถ้ากำหนด `allowedShifts` เอง ระบบจะเชื่อค่าที่ตั้งไว้
- ถ้า `allowedShifts` เป็น array ว่าง `[]` แปลว่า "ไม่รับเวรเลย"

### วันหยุด

- ใช้กำหนดวันหยุดราชการรายปี
- วันเสาร์-อาทิตย์ถือเป็นวันหยุดอยู่แล้วโดยอัตโนมัติ
- วันที่บันทึกไว้ในหน้านี้จะทำให้ระบบสร้าง slot แบบวันหยุดเพิ่มด้วย

### ตารางเวร

- `สร้างตารางเดือนใหม่` เพื่อเปิดหน้าของเดือนนั้น
- `Import Excel` ใช้ดึงตารางจากไฟล์ `.xlsx` กลับเข้าระบบ

### รายละเอียดตาราง

เมื่อเข้าหน้ารายเดือนแล้ว จะทำได้ดังนี้:

- `Generate ตาราง` สร้างตารางใหม่
- `Generate ใหม่` สุ่มและจัดใหม่ทั้งเดือน
- `Export Excel` ดาวน์โหลดไฟล์ตาราง
- `บันทึกยืนยัน` ล็อกตารางและส่งเข้า `ประวัติ`
- `ปลดล็อก` เปิดกลับมาแก้ไขได้อีก
- คลิกแต่ละช่องเพื่อเพิ่มหรือลบเวรของบุคคลนั้นในวันนั้น

## การนำเข้าข้อมูล

### Import สมาชิกจาก JSON

รองรับทั้งรูปแบบ:

- array ตรงๆ
- `{ "members": [...] }`
- `{ "data": [...] }`

ตัวอย่าง:

```json
[
  {
    "nickname": "สมชาย",
    "phaId": "PHA001",
    "dateOfBirth": "1990-03-15",
    "pregnancyStatus": null
  }
]
```

หมายเหตุ:

- `pregnancyStatus` ใช้ค่า `early`, `mid`, `late`, `postpartum`, หรือ `null`
- `nickname` หรือ `phaId` ที่ซ้ำจะถูกข้าม

### Import วันหยุดจาก JSON

รองรับทั้งรูปแบบ:

- array ตรงๆ
- `{ "holidays": [...] }`
- `{ "data": [...] }`

ตัวอย่าง:

```json
[
  { "date": "2026-01-01", "name": "วันขึ้นปีใหม่" },
  { "date": "2026-04-13", "name": "วันสงกรานต์" }
]
```

### Import ตารางจาก Excel

ข้อควรรู้:

- รองรับไฟล์ `.xlsx`
- ตารางของเดือนนั้นจะถูกแทนที่ทั้งหมด
- ระบบจะตั้งสถานะเป็น `ยืนยันแล้ว` อัตโนมัติ
- ถ้าในไฟล์มีชื่อที่ไม่ตรงกับสมาชิกในระบบ รายการนั้นจะถูกข้ามและแจ้งเป็น warning

```text
        ├── eligibility.ts  สิทธิ์เวรตามอายุ/ตั้งครรภ์
        ├── constraints.ts  HC1-HC4
        ├── solver.ts       greedy + backtracking
        ├── balance.ts      ปรับสมดุล workload
        ├── scoring.ts      soft constraints
        └── index.ts        generateSchedule()
```

## ปัญหาที่พบบ่อย

### เปิด Docker แล้วเข้าเว็บไม่ได้

ตรวจสอบว่า:

- เครื่องมี Docker ทำงานอยู่จริง
- port `3000` ไม่ถูกโปรแกรมอื่นใช้อยู่
- ใช้คำสั่ง `docker compose logs -f` แล้วไม่มี error ตอน start

### เปลี่ยนโค้ดแล้วหน้าเว็บยังไม่อัปเดต

ถ้าใช้ Docker image แบบในโปรเจกต์นี้ ให้ rebuild ใหม่:

```bash
docker compose up --build -d
```

### อยากล้างข้อมูลแล้วเริ่มใหม่ทั้งหมด

```bash
docker compose down -v
docker compose up --build
```

### กด Generate แล้วไม่ได้

ตรวจสอบว่า:

- มีสมาชิกที่ `เปิดใช้งาน` อยู่จริง
- มีวันหยุดของปีนั้นครบถ้วน
- ไม่มีทุกคนถูกจำกัดเวรจนระบบหา candidate ไม่ได้

### ระบบแจ้งว่าไม่มีคนทำเวรบางประเภทได้เลย

มักเกิดจาก:

- กลุ่มอายุสูงเกินไป
- ตั้ง `pregnancyStatus` จำกัดเวรไว้มาก
- ตั้ง `allowedShifts` แคบเกินไป

แนวทางแก้:

- เปิดใช้งานสมาชิกเพิ่ม
- ปรับ `allowedShifts`
- ตรวจข้อมูลวันเกิดและสถานะตั้งครรภ์

### Generate แล้วมี slot ว่าง

ระบบจะพยายามเติมให้มากที่สุดก่อน แต่ถ้ายังมี slot ว่าง ให้ลอง:

- เพิ่มจำนวนบุคลากรที่ active
- ผ่อนข้อจำกัดเฉพาะบุคคล
- กด `Generate ใหม่`

## ข้อควรรู้

- ระบบแสดงปีในหน้าเว็บเป็น `พ.ศ.` แต่จัดเก็บภายในเป็น `ค.ศ.`
- โปรเจกต์นี้ออกแบบให้ใช้งานแบบภายในเป็นหลัก
- ไม่มีระบบ authentication ในตัว ควรรันในเครื่องภายในหรือสภาพแวดล้อมส่วนตัว
