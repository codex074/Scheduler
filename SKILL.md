# Shift Scheduler — ห้องยาโรงพยาบาล

## ภาพรวม

Skill นี้ช่วยจัดตารางเวรห้องยาโรงพยาบาล โดยคำนึงถึง:
- ประเภทวัน (วันทำการ / วันหยุดเสาร์-อาทิตย์ / วันหยุดนักขัตฤกษ์)
- ผลัดเวรและแผนกที่ต้องครอบคลุม
- กฎการจัดเวรตามช่วงอายุบุคลากร
- ข้อจำกัดพิเศษของแต่ละเวร

## กฎ Hard Constraints (ห้ามละเมิดเด็ดขาด)

| # | กฎ | รายละเอียด |
|---|-----|-----------|
| HC1 | รุ่งอรุณ + ดึก | ห้ามคนเดียวกันรับเวรรุ่งอรุณ (07.00-08.30) และเวรดึก (00.00-08.30) ในวันเดียวกัน |
| HC2 | ดึก + เช้า | ห้ามคนเดียวกันรับเวรดึก (00.00-08.30) ต่อด้วยเวรเช้า (08.30-16.30) วันเดียวกัน |
| HC3 | บ่าย + ดึก | ห้ามคนเดียวกันรับเวรบ่าย (16.30-23.59) ต่อเนื่องกับเวรดึก (00.00-08.30) ของคืนเดียวกัน (ห้ามทำงานติดต่อกัน 16.30→08.30) |
| HC4 | ซ้อนทับเวลา | 1 คน รับได้ 1 เวรในช่วงเวลาเดียวกันเท่านั้น ห้ามเวรที่มีช่วงเวลาทับซ้อนกัน |
| HC5 | กฎตามช่วงอายุ | **[รอข้อมูล]** — จะเพิ่มเติมภายหลัง |

---

## กฎ Soft Constraints (พยายามทำตาม)

### ความหลากหลายของเวร
| # | กฎ | รายละเอียด |
|---|-----|-----------|
| SC1 | กระจายประเภทเวร | แต่ละคนควรได้รับเวรหลากหลายประเภท ไม่กระจุกอยู่เวรชนิดเดียวตลอดเดือน |
| SC2 | กระจายช่วงเวลาของเดือน | เวรไม่ควรกระจุกอยู่แค่ต้น/กลาง/ปลายเดือน ควรกระจายสม่ำเสมอตลอดเดือน |

### ความยุติธรรมวันหยุด
| # | กฎ | รายละเอียด |
|---|-----|-----------|
| SC3 | หมุนเวียนวันหยุดนักขัตฤกษ์ | ไม่ให้คนเดิมอยู่เวรวันหยุดนักขัตทุกครั้ง ควรหมุนเวียนให้ทั่วถึง |
| SC4 | กระจายตำแหน่งในวันหยุดยาว | ในช่วงวันหยุดยาวติดต่อกัน ไม่ให้คนเดิมอยู่เวรกลางช่วงวันหยุดตลอด ควรหมุนเวียนทั้งวันแรก กลาง และวันสุดท้ายของช่วงหยุด |
| SC5 | กระจายเวรวันเสาร์-อาทิตย์ | หมุนเวียนไม่ให้ใครได้วันหยุดสุดสัปดาห์น้อยกว่าคนอื่นมากเกินไป |

### ภาระงานโดยรวม
| # | กฎ | รายละเอียด |
|---|-----|-----------|
| SC6 | หลีกเลี่ยงเวรดึกติดต่อกัน | ไม่ควรจัดเวรดึกติดต่อกันหลายวันสำหรับคนเดียวกัน |
| SC7 | สมดุลจำนวนเวรรวม | จำนวนเวรรวมต่อเดือนของแต่ละคนไม่ควรต่างกันมากเกินไป |

## ข้อมูลที่ต้องรับจากผู้ใช้

เมื่อผู้ใช้ต้องการจัดตาราง ให้ถามข้อมูลต่อไปนี้หากยังไม่มี:

```
1. ช่วงเวลาที่ต้องการจัด (เช่น เดือนกรกฎาคม 2025)
2. รายชื่อบุคลากรพร้อม NickName, pha_id และวันเกิด
3. วันลา / วันหยุดพิเศษของแต่ละคน
4. วันหยุดนักขัตฤกษ์ในช่วงนั้น
```

---

## ขั้นตอนการจัดตาราง

### ขั้นที่ 1 — เตรียมข้อมูล
- รับรายชื่อบุคลากรพร้อม NickName, pha_id และวันเกิด
- คำนวณช่วงอายุของแต่ละคน ณ เดือนที่จัด
- ระบุวันทำการ / วันหยุด / นักขัตฤกษ์ในช่วงนั้น
- ระบุช่วงวันหยุดยาวติดต่อกัน (ถ้ามี)
- บันทึกวันลาหรือข้อจำกัดพิเศษของแต่ละคน

### ขั้นที่ 2 — กำหนด Slot ที่ต้องเติม
- สร้างรายการเวรทุกช่องที่ต้องมีคน ตามประเภทวัน
- ระบุเวรที่มีข้อจำกัดพิเศษ (SMC เฉพาะ จ-พฤ, รุ่งอรุณ ER เฉพาะ อ-ศ)

### ขั้นที่ 3 — จัดเวรตามกฎอายุ
- [รอข้อมูลกฎอายุ — จะเพิ่มเติมภายหลัง]

### ขั้นที่ 4 — ตรวจสอบ Hard Constraints
- ตรวจสอบ HC1-HC5 ทุกข้อ
- หากพบการละเมิด ให้แจ้งและปรับตารางใหม่

### ขั้นที่ 5 — ปรับให้ตรง Soft Constraints
- ตรวจสอบการกระจายเวรของแต่ละคน (SC1-SC2)
- ตรวจสอบความยุติธรรมวันหยุด (SC3-SC5)
- ตรวจสอบภาระงานโดยรวม (SC6-SC7)
- ปรับสลับเวรที่ยังกระจุกตัวอยู่

### ขั้นที่ 6 — สร้างไฟล์ Excel และสรุป
- สร้างไฟล์ Excel ตามรูปแบบที่กำหนด
- สรุปจำนวนเวรแต่ละประเภทของแต่ละคน
- แจ้งเตือนหากมีจุดที่ต้องตรวจสอบเพิ่มเติม

## รูปแบบ Output — ไฟล์ Excel

### โครงสร้างไฟล์
- **คอลัมน์ที่ 1** = NickName (ชื่อเล่นบุคลากร)
- **คอลัมน์ที่ 2** = pha_id (รหัสบุคลากร)
- **คอลัมน์ที่ 3 เป็นต้นไป** = วันที่ (1, 2, 3, ... ถึงสิ้นเดือน)
- **เซลล์** = สัญลักษณ์เวรที่ได้รับในวันนั้น

### ตารางสัญลักษณ์
| เวร | สัญลักษณ์ |
|-----|----------|
| บ่าย MED | บM |
| บ่าย ER | บE |
| โครงการพิเศษ | Ext |
| SMC | SMC |
| รุ่งอรุณ OPD | รO |
| รุ่งอรุณ ER | รE |
| ดึก | ด |
| เช้า SURG | s |
| เช้า MED (D/C) | d |
| เช้า MED (Cont) | c |
| เช้า ER | e |

### กฎการแสดงผล
- ถ้า 1 คนมีมากกว่า 1 เวรในวันเดียวกัน ให้ใช้ `/` คั่น เช่น `รO/SMC` หรือ `รE/Ext`
- เซลล์ว่าง = ไม่มีเวร

### ตัวอย่าง
```
| NickName  | pha_id | 1   | 2      | 3  | 4   | 5  |
|-----------|--------|-----|--------|----|-----|----|
| พี่ใจ     | PHA0   | บM  |        | ด  |     | บE |
| พี่หนึ่ง  | PHA51  |     | รO/SMC |    | s   |    |
| พี่เปรียว | PHA12  | บE  |        |    | d   | ด  |
```
